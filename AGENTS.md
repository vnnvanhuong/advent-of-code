## Process to solve a problem

Read the problem defined in the @aoc2025 directory, but do not implement the solution until I confirm it. Use the practices defined in the @docs directory. The solution should be in Golang. Please refer to existing problems, tests, implementation, and walkthrough documents if any. Here is the process I want to follow, step by step:

- Step 1: Summarize the problem in simple words to ensure understanding.
- Step 2: Write out some test cases to demonstrate the input and output.
- Step 3: Come up with a logical solution first, sometimes guided by intuition.
- Step 4: Dry-run the proposed solution with the defined test cases.
- Step 5: Code the solution and calculate its time and space complexity.
- Step 6: Propose approaches for optimization (if any)
- Step 7: Recode (or refine) the solution and recalculate time and space complexity (if any)
- Step 8: Summarize the takeaway or lessons learned (e.g., algorithmic themes or well-known algorithms) from the solved problem.

Remember to update the walkthrough document at each step, but do not add real code into it; use pseudocode instead. Read the problem input from the .txt file in @aoc2025, and print the result in the main program (main.go), commenting out other problems to avoid unnecessarily executing all problems. Mark the result as redacted in the walkthrough document.
